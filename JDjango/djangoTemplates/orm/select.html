1. 单一对象
    one_entry = ${model_name}.objects.get(pk=1)
    符合条件的值必须有且只有一个，否则会抛异常
  2. 全部对象
    all_entries = ${model_name}.objects.all()
    all_entries是可迭代对象
  3. 符合条件【WHERE】
    filter_entries = ${model_name}.objects.filter(...)
  4. 不符合条件【WHERE NOT】
    exclude_entries = ${model_name}.objects.exclude(...)
    exclude 参数使用参考 filter ，一模一样，不同的是查询结果完全相反
  5. 排序【ORDER BY】
    order_entries = ${model_name}.objects.order_by('-pub_date') order_entries = ${model_name}.objects.order_by('?')
    1、按照模型 Meta 中的 ordering 选项给出的排序元组排序
    2、默认升序，前缀'-'表示倒序，'?'表示随机排序，不推荐随机排序
  6. 新字段以及聚合 【GROUP BY】
    obj = ${model_name}.objects.annotate(number_of_entries=Count('entry')) number_of_entries = obj[0].number_of_entries
    1、from django.db.models import Count
    2、entry 可以是外键在 ${model_name} 中的属性名
    3、只有引用单个字段的聚合表达式才能成为位置参数，其他一切都必须是关键字参数
  7. 切片【slice】
    cut_entries = ${model_name}.objects.all()[:5]
    和Python切片语法不同之处在于Django不允许负索引！
