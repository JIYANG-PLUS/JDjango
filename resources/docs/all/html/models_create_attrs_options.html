Django模型字段选项

什么是字段选项？

字段选项是模型类的属性，也可以理解为数据库表字段的属性。

模型的字段选项，主要用于设置数据库表字段的长度、是否为空、默认值、帮助文本、主键、唯一性等属性。

大部分的字段选项是所有字段共享的，所以这里统一罗列出来。

可选的字段选项有哪些？

参数名称参数含义值类型默认值示例代码
blank 字段允许为空 布尔型 False blank=True
null blank=True且字段为空时数据库默认赋值NULL 布尔型 False null=True

unique_for_date 日期组合唯一 字符型 unique_for_date='日期字段名'

choices 可选列表 二元组 choices=[('存储值','展示值'),]

注意点：

  1. 尽量不要在字符串类型的字段上使用null=True参数设置，因为这会使字符型字段产生两种可能值：NULL和空字符串。
  2. 字符串类型的字段同时设置unique=True和blank=True时，必须设置null=True。
  3. 一个模型必须且只能拥有一个显式主键设置，primary_key=True还可以用null=False和unique=True隐式指定，效果等同。
  4. 在多对多和一对一关系字段中不允许使用unique参数。
  5. 当unique=True时，不必再指定db_index=True，因为unique=True默认实现了db_index=True。
  6. 不允许在参数赋值中使用lambda表达式，lambda不会被序列化迁移！
  7. 凡是【参数含义】以“表单”开头的参数，与表字段的创建无关，只与前台展示的表单有关。
  8. 关于表单的提示信息若涉及HTML字符，推荐使用django.utils.html.escape()转义HTML字符串。
